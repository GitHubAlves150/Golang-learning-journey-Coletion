🐙 Git e GitHub - Guia Completo

Comandos essenciais para o dia a dia do desenvolvedor

📥 1. Instalação do Git

🔹 Windows

Baixar em: https://git-scm.com/download/win
Instalar com as opções padrão

🔹 Linux (Ubuntu/Debian)

sudo apt update
sudo apt install git
git --version  # Verificar instalação

🔹 macOS

brew install git
# ou baixar em https://git-scm.com/download/mac

⚙️ 2. Configuração Inicial

ComandoDescrição
git config --global user.name "Seu Nome"Configura nome do usuário
git config --global user.email "seu@email.com"Configura email do usuário
git config --listLista todas configurações
git config --global core.editor "code --wait"Configura VS Code como editor padrão
💡 Dica: Use --global para aplicar em todos repositórios ou remova para configurar apenas no repositório atual.

🔑 3. Configurar Chave SSH (autenticação sem senha)

🔹 Gerar chave SSH

# Gerar chave (recomendado: ed25519)
ssh-keygen -t ed25519 -C "seu@email.com"

# Aperte Enter para local padrão
# Digite uma senha (opcional) ou Enter para sem senha

🔹 Verificar chave gerada

ls -la ~/.ssh/
# Deve aparecer id_ed25519 (privada) e id_ed25519.pub (pública)

🔹 Copiar chave pública

cat ~/.ssh/id_ed25519.pub
# Copie todo conteúdo que aparecer

🔹 Adicionar no GitHub

1. Acesse: GitHub → Settings → SSH and GPG keys
2. Clique em "New SSH key"
3. Title: "Meu computador"
4. Key: Cole a chave copiada
5. Add SSH key

🔹 Testar conexão

ssh -T git@github.com
# Deve aparecer: "Hi Usuario! You've successfully authenticated..."
💡 Com SSH configurado, você não digita usuário ou token nunca mais!

📁 4. Comandos Básicos do Git

ComandoDescrição
git initInicializa repositório Git na pasta atual
git statusVerifica status dos arquivos (modificados, adicionados, etc.)
git add arquivo.txtAdiciona arquivo específico para commit
git add .Adiciona TODOS os arquivos modificados
git add -AAdiciona todos (incluindo deletados)
git commit -m "mensagem"Cria commit com mensagem
git commit --amend -m "nova msg"Corrige mensagem do último commit
git log --onelineMostra histórico resumido
git log --graph --allMostra histórico em formato gráfico

🌐 5. Sincronizar com GitHub (Remoto)

🔹 Clonar repositório existente

# HTTPS (pede token)
git clone https://github.com/usuario/repositorio.git

# SSH (recomendado - sem senha)
git clone git@github.com:usuario/repositorio.git

🔹 Conectar repositório local ao remoto

git remote add origin https://github.com/usuario/repositorio.git
# ou SSH
git remote add origin git@github.com:usuario/repositorio.git

🔹 Verificar conexões remotas

git remote -v

🔹 Trocar de HTTPS para SSH

git remote set-url origin git@github.com:usuario/repositorio.git

🔹 Enviar para GitHub

# Primeira vez (configura upstream)
git push -u origin main

# Próximas vezes
git push

# Enviar branch específica
git push origin nome-branch

🔹 Baixar do GitHub

git pull              # Busca e mescla
git fetch             # Só busca, não mescla
git fetch --prune     # Remove referências de branches deletadas

🌿 6. Branches (Ramos)

ComandoDescrição
git branchLista branches locais
git branch -aLista todas branches (local + remoto)
git branch -rLista apenas branches remotas
git checkout -b nova-branchCria e muda para nova branch
git checkout mainMuda para branch main
git switch mainMuda para branch main (versão moderna)
git merge branchMescla branch atual com outra
git branch -d branchDeleta branch local
git push origin --delete branchDeleta branch remota

⏪ 7. Desfazer/Reverter Alterações

ComandoDescrição
git restore arquivo.txtDesfaz alterações (antes do add)
git restore --staged arquivo.txtRemove arquivo do stage (desfaz add)
git reset HEAD~1Desfaz último commit (mantém alterações)
git reset --hard HEAD~1Desfaz commit E perde alterações ⚠️
git reset --hard origin/mainReseta para estado do GitHub ⚠️
⚠️ Cuidado! Comandos com --hard são irreversíveis. Use com atenção!

📦 8. Stash - Guardar trabalho temporariamente

ComandoDescrição
git stashGuarda alterações temporariamente
git stash listLista todos stashes salvos
git stash popRecupera último stash e remove da lista
git stash applyRecupera stash mas mantém na lista
git stash dropRemove último stash
💡 Use stash quando: Precisa mudar de branch urgente mas não quer commitar código incompleto.

👁️ 9. Ver diferenças (Diff)

ComandoDescrição
git diffMostra diferenças nos arquivos (antes do add)
git diff --stagedMostra diferenças após add (antes do commit)
git diff main..branchDiferença entre duas branches
git showMostra detalhes do último commit
git show hashMostra detalhes de commit específico

🔐 10. Gerar Token GitHub (para HTTPS)

1. GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)
3. Generate new token (classic)
4. Note: "git push"
5. Expiration: 90 dias (ou personalizado)
6. Scopes: Marcar "repo"
7. Generate token
8. COPIE O TOKEN (ex: ghp_abc123...)
💡 O token é usado como senha quando o Git pedir autenticação via HTTPS.

⚡ 11. Fluxo de Trabalho Diário (Resumo)

# 1. Baixar alterações
git pull

# 2. Verificar status
git status

# 3. Ver o que mudou
git diff

# 4. Adicionar tudo
git add .

# 5. Commitar
git commit -m "descrição do que fez"

# 6. Subir para GitHub
git push

# 7. Ver histórico
git log --oneline --graph --all

🐛 13. Erros Comuns e Soluções

ErroSolução
Permission denied (publickey)Chave SSH não configurada ou não adicionada no GitHub
Authentication failedUsar TOKEN (não senha) no HTTPS ou configurar SSH
Failed to pushFazer git pull primeiro para baixar alterações
Merge conflictResolver conflitos manualmente, depois git add . e git commit
detached HEADVoltar com git checkout main ou git switch main

🚀 14. Criar Aliases (Atalhos para comandos)

# Criar alias "historico" para ver log bonito
git config --global alias.historico "log --oneline --graph --all"

# Usar:
git historico

# Criar alias "tudo" para ver status + log + branches
git config --global alias.tudo '!git status && git log --oneline -5 && git branch -a'

# Criar alias "ultimo" para ver último commit
git config --global alias.ultimo "log -1 --stat"
